Виконання лабораторної роботи передбачає вивчення прийомів розробки REST API за допомогою сучасних PHP-фреймворків, а також закріплення навичок роботи з системою контролю версій Git:

* Створення та налаштування REST-маршрутів;
* Реалізація повного CRUD-циклу для сутностей бази даних;
* Організація роботи з локальною базою даних SQLite;
* Документування та тестування ендпоінтів через Postman колекції;
* Виконання комітів та вивантаження локального репозиторію на GitHub.

## У лабораторній роботі було:
1. Створено та налаштовано backend-додаток на фреймворку **Laravel** (порт `8000`), де за допомогою Eloquent ORM реалізовано CRUD для сутності `Playlist`.
2. Створено та налаштовано backend-додаток на фреймворку **Symfony** (порт `8081`), де за допомогою Doctrine ORM та сутностей (Entities) реалізовано аналогічний CRUD для `Playlist`.
3. Створено та повністю задокументовано Postman-колекцію `frameworks_playlist_api.json` з детальними описами кожного ендпоінту для перевірки працездатності обох API.
4. Ініціалізовано спільний Git-репозиторій для обох фреймворків та завантажено фінальний чистий код на GitHub.

## Використані команди
Під час виконання лабораторної роботи було використано такі команди:
* **Команди Laravel:** `php artisan migrate`, `php artisan serve`, `composer install:api`.
* **Команди Symfony:** `php bin/console make:entity`, `php bin/console make:migration`, `php bin/console doctrine:migrations:migrate`, `php bin/console make:controller`, `php -S 127.0.0.1:8081 -t public`.
